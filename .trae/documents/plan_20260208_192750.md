# Plano de Melhoria do Sistema de Notificações com Expo Notifications

Vou implementar uma solução completa e robusta para o sistema de notificações, focada em confiabilidade, personalização e independência de serviços externos (exceto Expo Notifications).

## 1. Refatoração do Serviço de Notificações (`services/notifications.ts`)
Aprimorarei o serviço existente para incluir:
- **Sistema de Templates**: Centralizar as mensagens em templates reutilizáveis e personalizáveis.
- **Lógica de Agendamento Inteligente**:
  - Garantir identificação precisa de vencimentos nos próximos 7 dias.
  - Implementar cálculo de datas robusto (considerando anos bissextos e fins de mês).
  - Adicionar suporte a notificações recorrentes de forma eficiente.
- **Mecanismo de Retry**: Implementar função de tentativa automática para falhas no agendamento (`scheduleNotificationWithRetry`).
- **Independência e Robustez**:
  - Isolar o log do Firebase em um `try/catch` para que falhas de rede não impeçam o agendamento local.
  - Adicionar logs locais detalhados para debug.
- **Permissões**: Melhorar o fluxo de verificação e solicitação de permissões.

## 2. Implementação de Testes Unitários (`services/__tests__/notifications.test.ts`)
Criarei um arquivo de testes para validar:
- O cálculo correto das datas de vencimento.
- A geração das mensagens a partir dos templates.
- A lógica de retry e tratamento de erros.
*Nota: Será necessário configurar o Jest no projeto para executar estes testes.*

## 3. Validação e Integração
- Verificarei a integração com `RecurrenceView.tsx` para garantir que as alterações não quebrem a funcionalidade existente.
- Garantirei que o sistema opere 100% localmente para o agendamento, usando o Expo Notifications.
